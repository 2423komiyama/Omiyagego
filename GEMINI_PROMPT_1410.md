# Omiyage Go データ収集プロンプト（47都道府県 × 30件 = 1,410件）

## 依頼内容

日本の旅行者向けお土産情報サービス「Omiyage Go」のデータベース構築のため、以下の仕様に従って47都道府県のお土産データを生成してください。

## 対象都道府県と収集件数

**各都道府県30件**を収集してください。以下の内訳を守ること：

- **定番銘菓・グルメ**：15件（全国的に有名な商品）
- **地元民のみぞ知るニッチなソウル土産**：10件（地元スーパー・道の駅・地元商店でしか買えないもの）
- **観光地限定品**：5件（特定の観光スポット・施設でのみ購入可能）

## フィールド定義

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| id | string | ✅ | `p-{都道府県ローマ字}-{3桁連番}` 例: `p-hokkaido-001` |
| name | string | ✅ | 商品名（日本語） |
| brand | string | ✅ | ブランド・メーカー名 |
| description | string | ✅ | 商品説明（100〜200文字） |
| price | number | ✅ | 税込価格（円） |
| imageUrl | null | ✅ | 常に `null`（画像URLは別途設定） |
| prefecture | string | ✅ | 都道府県名（例: "北海道", "東京都", "大阪府"） |
| region | string | ✅ | 地方名（北海道/東北/関東/中部/近畿/中国/四国/九州・沖縄） |
| category | string | ✅ | カテゴリ（下記リスト参照） |
| shelfLife | number | ✅ | 賞味期限（日数） |
| isIndividualPackaged | boolean | ✅ | 個包装かどうか |
| servingSize | number | ✅ | 個数・容量 |
| guaranteeReason | string[] | ✅ | 「外さない理由」3〜5項目 |
| makerStory | string | ✅ | メーカー・商品の背景ストーリー（50〜150文字） |
| badges | string[] | ✅ | バッジ（下記リスト参照） |

## バッジ一覧

```
bestseller   - 売上上位

local        - 地元産・地元限定
limited      - 期間・数量限定
new          - 新商品
organic      - オーガニック・自然素材

soul         - ニッチなソウル土産（地元民御用達）
```

## 品質要件

1. **実在する商品のみ**：架空の商品は禁止。実際に販売されている商品のみ記載。
2. **価格の正確性**：2026年3月現在の実勢価格を記載。
3. **多様性の確保**：
   - 価格帯：500円〜10,000円まで幅広く
   - カテゴリ：各都道府県で最低5カテゴリ以上
   - バッジ：`soul`（ニッチなソウル土産）を各都道府県10件以上に付与
4. **説明文の質**：旅行者が「これを買いたい」と思える具体的な説明を記載。
5. **地域性の反映**：その地域ならではの特産品・食文化を反映させること。

## 出力方法

**都道府県ごとに分割して出力**してください（一度に全件出力するとトークン制限に引っかかるため）。

### 出力順序
1. 北海道（30件）
2. 青森県（30件）
3. 岩手県（30件）
...以下47都道府県まで続ける

### 各出力のフォーマット
```json
{
  "prefecture": "北海道",
  "products": [
    { ...商品データ... },
    ...30件
  ]
}
```

---

## 注意事項

- コメント（`//`）は含めないこと（JSONパースエラーの原因になる）
- 末尾カンマは含めないこと
- 文字列内の改行は `\n` でエスケープすること
- `imageUrl` は常に `null` にすること（後から設定するため）
- `id` は重複しないよう、都道府県ローマ字 + 3桁連番で統一すること
